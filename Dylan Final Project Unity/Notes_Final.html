Episode 8: Game Over

*Making a "Game Over" screen or even a "Main Menu" should be done in a "Game Manager

-To start this, you create an empty object in the "Hierarchy View" and reset its "Transform"

-Add a new component as a "Script" labeled "Game Manager"

*The Following notes will detail the process of creating a "Game Over" by having the level restart upon failure.

- First, delete the two standard functions (Void Start and Void Update)

-Now create a new "Custom Function" called, "public void EndGame () { }"

Note: adding "public" to the void makes it accessible from other scripts since without it, its considered "private" to that specific script.

*In order to relate this to the player object we can go into the "Player Collision" Script, and under the if statement enter, "FindObjectOfType().endGame();"

-This same code can be added in an "if" statement in the "Player Movement" script for when the player falls off the map.

*If you want to restart your game on a "Game Over" or even change the scene depending on what events occur you will need to add "using UnityEngine.SceneManagement;" at the top of your script.

*In order to restart a level on the same scene use "SceneManager.LoadScene(SceneManager.GetActiveScene().name);"

-Restarting the game with this is initially quite jarring since the scene instantly resets itself so adding a delay can help smoothen the restart.

-Using "Invoke" is a good way of doing this.

*"File" -> "Build Settings"

-This will show all the scenes that will show up when you export the game and lines them up in a queue.

Episode 9: Winning the Level

*The Purpose of this part is creating a simple level win/complete screen and making it appear after reaching the end of the level

* To start we can create a "Cube" object and reset its transform

- This will be used as the sort of passing point for when the level is supposed to end.

- Resize to be big enough that when the player reaches the area the trigger area will activate

-We turn off the "Mesh Renderer" so that it does not appear in the game.

* Activate the "Is Trigger" box in the "Inspector View"

* We will then create a script named something like "EndTrigger" after placing it in the position in which we want our game to end.

- After removing the existing "Voids" create an "OnTriggerEnter" void

-Also create a public void that links to the "GameManager" and rename it to something similar like "gameManager"

*Now on the topic of the Game Manager, we move to its specific script and add a public void for completing the level and we can call it "LevelComplete"

-Go back to the "OnTriggerVoid" in the other script and enter "gameManager.LevelComplete();"

*Make sure to add the "GameManager"(not the script) into the "EndTrigger" script

-Go to your "Canvas" next or create it in the "Hierarchy View"

-Create a "Panel" as the base for you win screen

*After creating your win screen create a public game object in the "GameManager" script to place it into, and reference it in the public void you made there.

-Enter into that void, "completeLevelUI.SetActive(true);"

*Next some animation can be added to make the transition into the wind screen much smoother

-First go to the "Window" tab and select "Animation" and select the option again within the drop off menu

-You can adjust with the time sliders, where certain events will take place such as the panel slowly fading into view

*After completing the level, you may want to load the next one right, after and this is where you can add "Script Events" to the animation reel

-Go to your win screen panel that you created and make a new script

*Make sure you add "using UnityEngine.SceneManagement;" at the top of the code since you will be working with the scenes.

-Create a public void and within the brackets enter, "SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex + 1);"

note: This will attain the current scene and then move to the next one in the "Build Settings" queue

Final Episode: Finishing Up

Note: To cleanly duplicate levels and still be able to change settings on the objects make important things like the "Game Manager" "Player" and "Main Camera" into Prefabs

*To create a Welcome and End Screen, we create new scenes

-Creating these screens are as simple as creating a panel and text to go with it, but adding buttons can help either exit out of the game or start it.

*Whether it be in your panel, text, or the button itself; create a script

-In it, add a public void for exiting the game

-Within the brackets enter "Application.Quit();" to have the button close the game

Note: A similar process can be done for the start button on the welcome page

-We can enter into the new public void of this new script, "SceneManager.LoadScene(SceneManager.GetActiveScene().buildIndex + 1);"

-This will load the next level/first level (This depends on how the queue is set up in the build settings)
