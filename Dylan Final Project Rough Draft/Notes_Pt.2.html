Episode 4: Camera Follow

- The simplest way to get the camera to follow the player is to "Parent" the camera to the player by dragging the camera onto the player in the "Hierarchy View"

Note: using this method will cause the camera will rotate with the player

*The other way to have the camera follow the player is to create a script for the camera

-Enter in the code "public Transform player;" and drag the player object from the hierarchy view into the new "player" slot in the camera's "Inspector View"

Note: â€œTransform" and "transform" are different

Transform: Refers to another object

transform: Refers to the current object

-the code "transform.position = player.position;" will cause the camera to follow the player

Note: Leaving the code at this will cause the camera to be placed inside of the player object, creating a sort of "First Person" perspective

*The Vector3 variable stores 3 floats and can be used to specify the "x, y , and z" values

*When creating the "Vector3" and naming it (ex: offset") you will take the code for "transform.position" and change it to "transform.position = player.position + offset;"

Episode 5: Collision

*when creating code for object collision you can remove the starting "void start" and "void update" methods

Note: when creating collision code, you can add it to the existing code for movement or create a new script for it

*The method for collision is "void OnCollisionEnter ()"

-This means that whatever code is within the brackets, will occur when this object collides with something. (needs "RigidBody" and "Collider")

-you can find objects by using collisionInfo.gameObject.name == "insert object name here"

note: this can cause complications like slow down the more objects that are added to the game

*instead of "name" using "tag" can put objects into categories and makes the search process faster and easier

-to create a new tag for an object or set of objects

*If you want an object to stop moving after it collides with something you can make a variable based on the created movement script

-After dragging the movement script into the new slot (ex: movement) you can enter within the if statement that says, "When this object hits object2...." you can enter

movement.enabled = false;

Episode 6: Gameplay

you can drag objects like your Obstacles into the "Project View" in order to turn them into reusable assets

In the "Scene View" after selecting an object you can select an axis from the top left of the view to get better angles for designing

-You can add a new layer to an object and after selecting it, lock it

This will make selecting other objects much easier, as locking it prevents it from being selected in the "Scene View"

*If you want to be more precise you can go to "Edit" and select "Snap Settings"

-Snap Settings allow movement only along a grid and the length of that movement can be determined by editing how for each goes in the x,y,z panel that pops up after selecting the setting

-You can press ctrl + d in order to copy an object in the scene view

*to use the snap setting, hold ctrl when moving the object

-Simply adding force to an object builds up momentum all the way up to the designated force

Force Types

-This comes in at "x, y, z, Force Type" when adding force

-"Velocity Change" completely ignores the mass of the object when applying force

*If you move to the "Window" tab and then "Rendering" tab, you can find the Lighting settings

-there, you can adjust lighting, shadows, add fog, and more

-increasing the "Drag" increases the air resistance the object faces when moving allowing for more responsive movement.

Episode 7: Score and UI

Build Settings Window:

*To begin, go to the "Hierarchy View", right click, "Create", "UI" and select what you would want

-You can go into the 2D View to more easily customize your UI

-If you want text in your UI to resize change the setting on the "Canvas" labeled "UI Scale Mode" from "Constant Pixel Size" to "Scale With Screen Size"/p>

-Also create a variable that will contain the "Score"

*ex: scoreText.text = player.position.z.ToString("0");

-the "("0")" is so that the score is only counted in whole numbers and not excessive decimal places

*Because this is interacting with your new "UI" that was created; at the top of the script, add "using UnityEngine.UI;"
