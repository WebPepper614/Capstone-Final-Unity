Video Tutorial

The online video tutorial im following goes through all the basics of general game creation within the unity engine, and goes through the creation of a "Endless Runner" as the game example.

Episode 1: Basics

-When creating a new project, you can choose between making it 2D or 3D (Only a few settings are changed between the two, and those changes can be altered later.)

*If you move over to the "Hierarchy view" you can right click inside of it(or click on create) to create objects such as squares or spheres as an example

-Going to "Transform" on an object and hitting reset, will center it in the scene view

*In order to give color to an object you can create and assign a material to it

-To create a material, you go to the "Project View", right click, and create material

-A good habit would be to use naming conventions for these such as playerMat

*In order to give an object gravity and actual physics, you must a "Collider", "Mesh Renderer", and the "Rigid Body" component

Episode 2: Programming

*To start, you must go to "Add Component", "Add Script"

-side note: you can drag components from the "Project View" into the "Inspector View" of other objects to add it to them

-Delete The top two "using tags" to clean up

-"Void Start" means that whatever code is within the brackets, will be executed as soon as the game starts.

-after framing the selected object (F) click on the "Move Tool" in the top left of the toolbar (Shortcut is "W"). You can then drag the object using the arrows (red= left/right) (green= up/down)

-Use Capitals in C# as it is case sensitive

-To write messages into the console type "Debug.Log()"

* Enter in "Public" above the brackets and enter in a component and its reference name

Ex: public RigidBody rb;

This will add an empty slot under your created script

-You can then drag the "RigidBody" component into that slot, so whenever you reference "rb" in the code it references "RigidBody"

-When entering code such as "rb.AddForce(); you need to specify details within the parenthesis

-When specifying the axis "(0,0,0) it goes "x,y,z"

* multiplying by Time.deltaTime essentially evens things out no matter the frame rates your computer is running

*When calculating physics (Adding forces, changing velocity, etc.) itâ€™s better to use "FixedUpdate" rather than just "Update"
